# Documentation for Setup and Testing

This repository contains a production-ready **Task Management System** built with **FastAPI** (Backend) and **React** (Frontend). It demonstrates secure authentication, role-based access control (RBAC), and scalable architecture practices.

### Backend (FastAPI)
- **Authentication**: JWT-based secure login & registration (BCrypt hashing).
- **RBAC**: Role-based access control (Admin vs. User).
- **CRUD Operations**: Complete management for Tasks.
- **Security**: Input validation (Pydantic), exception handling, and secure headers.
- **Database**: PostgreSQL with SQLAlchemy ORM (Async).
- **Deployment**: Dockerized services for easy setup.

### Frontend (React + Vite)
- **User Interface**: Clean UI for Login, Registration, and Dashboard.
- **Integration**: Connects seamlessly with the backend APIs.
- **State Management**: Handles user sessions and task data.

## ğŸƒâ€â™‚ï¸ Getting Started

### Prerequisites
- Docker & Docker Compose installed.
- **OR** Python 3.11+ and Node.js for local manual setup.

### Option 1: Run with Docker (Recommended)
1. **Clone the repository**:
   ```bash
   git clone https://github.com/dev-m03/prime-trade-fullstack
   cd PrimeTrade
   ```

2. **Set up environment variables**:
   ```bash
   copy .env.example .env
   # Edit .env if needed (default values work for local dev)
   ```

3. **Build and Run**:
   ```bash
   docker-compose up --build
   ```

4. **Access the Application**:
   - **Backend API**: [http://localhost:8000](http://localhost:8000)
   - **API Documentation (Swagger)**: [http://localhost:8000/docs](http://localhost:8000/docs)
   - **Frontend**: [http://localhost:5173](http://localhost:5173) (or check console for port)

### Option 2: Run Locally (Manual)

#### Backend
1. Create a virtual environment:
   ```bash
   python -m venv venv
   source venv/bin/activate  # Windows: venv\Scripts\activate
   ```
2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
3. Run the server:
   ```bash
   uvicorn app.main:app --reload
   ```

## Frontend to run seperately (Not dockerised)
1. Open new terminal and navigate to frontend directory:
   ```bash
   cd frontend
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Run:
   ```bash
   npm run dev
   ```

## ğŸ—ï¸ Scalability & Design Note

To ensure the system addresses the "Scalability" requirement, the following design decisions were made:

1.  **Stateless Authentication (JWT)**: The backend uses JSON Web Tokens (JWT) for authentication. This is purely stateless, allowing the backend to scale horizontally across multiple instances without needing shared session storage (like a sticky session).
2.  **Async Database Access**: We utilize `AsyncSQLAlchemy` with `asyncpg`. This non-blocking IO approach prevents the main thread from waiting for database queries, significantly increasing the throughput for high-concurrency requests compared to synchronous drivers.
3.  **Modular Architecture**: The codebase is structured purely by domain components (`auth`, `tasks`). This "Modular Monolith" approach makes it easy to extract specific modules into separate Microservices in the future if a specific domain (e.g., Task Processing) requires independent scaling.
4.  **Containerization**: The entire stack is Dockerized. This ensures consistency across environments and allows for orchestration via Kubernetes or AWS ECS for auto-scaling based on load.
5.  **Future Improvements**:
    - **Caching**: Redis could be added to cache frequent `GET` requests (e.g., getting task lists) to reduce DB load.
    - **Load Balancing**: Nginx can be placed in front of multiple FastAPI containers to distribute traffic.
